Справка по системе StudentRequest

Назначение системы

StudentRequest предназначена для приема, учета и обработки заявок студентов в структурном подразделении Университета. Система позволяет хранить обращения в единой базе данных, отслеживать статусы обработки и фиксировать комментарии ответственных сотрудников.

Студент

Пользователь с ролью студента может создавать заявки, выбирать категорию обращения, прикреплять файлы, просматривать статус обработки и читать комментарии сотрудника.

Сотрудник

Сотрудник подразделения просматривает поступившие заявки, принимает их в работу, изменяет статус, добавляет комментарии и закрывает обработанные обращения.

Администратор

Администратор управляет пользователями, категориями заявок, настройками системы, а также просматривает отчеты и журнал действий.

Основной порядок работы

  1. Пользователь входит в систему под своей учетной записью.
  2. Студент создает заявку и указывает тему обращения.
  3. При необходимости студент прикрепляет файл к заявке.
  4. Сотрудник подразделения просматривает новую заявку.
  5. Сотрудник меняет статус заявки и добавляет комментарий.
  6. После обработки заявка получает итоговый статус.
  7. Администратор может сформировать отчет по заявкам.

Статусы заявок

  • Создана — заявка отправлена студентом и ожидает рассмотрения.
  • Принята в работу — сотрудник начал обработку обращения.
  • Требует уточнения — студенту необходимо предоставить дополнительные сведения.
  • Обработана — сотрудник выполнил необходимые действия по заявке.
  • Закрыта — работа по заявке завершена.
  • Отклонена — заявка не может быть обработана по указанной причине.

Тестовые учетные записи

Для проверки работы системы используются демонстрационные учетные записи с ролями студента, сотрудника и администратора. Они предназначены для демонстрации основного сценария работы приложения.
