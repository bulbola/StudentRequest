# StudentRequest

**StudentRequest** — учебная веб-информационная система учета и обработки заявок студентов в структурном подразделении Университета.

Проект подготовлен для преддипломной практики по теме:

> Разработка веб-информационной системы учета и обработки заявок студентов в структурном подразделении Университета

## Назначение системы

Система предназначена для централизованного приема, учета и обработки обращений студентов. Она позволяет студентам создавать заявки, сотрудникам подразделения обрабатывать обращения, а администратору управлять пользователями, категориями и отчетностью.

## Основные возможности

* авторизация пользователей;
* разделение доступа по ролям;
* личный кабинет студента;
* панель сотрудника подразделения;
* административная панель;
* создание и просмотр заявок;
* изменение статусов заявок;
* добавление комментариев к заявкам;
* прикрепление файлов;
* просмотр истории обработки;
* формирование отчета по заявкам;
* справка по системе.

## Роли пользователей

### Студент

Студент может создавать заявки, выбирать категорию обращения, прикреплять файлы, просматривать статус обработки и читать комментарии сотрудника.

### Сотрудник подразделения

Сотрудник может просматривать поступившие заявки, принимать их в работу, изменять статусы, добавлять комментарии и закрывать обработанные обращения.

### Администратор

Администратор может управлять пользователями, категориями заявок, настройками системы, отчетами и журналом действий.

## Локальный запуск

### 1. Создать виртуальное окружение

```bash
py -m venv .venv
```

### 2. Активировать виртуальное окружение

Для Windows PowerShell:

```powershell
.venv\Scripts\activate
```

### 3. Установить зависимости

```bash
py -m pip install -r requirements.txt
```

### 4. Создать базу данных

```bash
py -m flask --app run.py init-db
```

### 5. Заполнить базу тестовыми данными

```bash
py -m flask --app run.py seed-db
```

### 6. Запустить приложение

```bash
py -m flask --app run.py run
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5000
```

## Основной сценарий демонстрации

1. Войти в систему под учетной записью студента.
2. Создать новую заявку.
3. При необходимости прикрепить файл.
4. Выйти из системы.
5. Войти под учетной записью сотрудника.
6. Открыть заявку студента.
7. Изменить статус заявки.
8. Добавить комментарий.
9. Войти под учетной записью администратора.
10. Проверить пользователей, категории и отчетность.

## Статусы заявок

* **Создана** — заявка отправлена студентом.
* **Принята в работу** — сотрудник начал обработку заявки.
* **Требует уточнения** — студенту необходимо предоставить дополнительные сведения.
* **Обработана** — сотрудник выполнил необходимые действия.
* **Закрыта** — работа по заявке завершена.
* **Отклонена** — заявка не может быть обработана.
