# Ручные тестовые сценарии

Документ содержит набор ручных тест-кейсов для проверки StudentRequest.

## TC-01. Открытие главной страницы

**Шаги:**

1. Запустить приложение.
2. Открыть адрес `http://127.0.0.1:5000`.

**Ожидаемый результат:**  
Открывается главная страница системы с описанием ролей и кнопкой входа.

## TC-02. Вход под студентом

**Шаги:**

1. Открыть страницу входа.
2. Ввести `student@example.com`.
3. Ввести пароль `student123`.
4. Нажать кнопку входа.

**Ожидаемый результат:**  
Открывается личный кабинет студента.

## TC-03. Создание заявки студентом

**Шаги:**

1. Войти под студентом.
2. Нажать кнопку создания заявки.
3. Выбрать категорию.
4. Заполнить тему и описание.
5. Выбрать приоритет.
6. Нажать кнопку создания заявки.

**Ожидаемый результат:**  
Новая заявка появляется в списке заявок студента.

## TC-04. Вход под сотрудником

**Шаги:**

1. Выйти из учетной записи студента.
2. Войти под сотрудником.
3. Открыть панель сотрудника.

**Ожидаемый результат:**  
Сотрудник видит очередь заявок.

## TC-05. Обработка заявки сотрудником

**Шаги:**

1. Открыть заявку из очереди.
2. Выбрать новый статус.
3. Ввести комментарий.
4. Сохранить изменения.

**Ожидаемый результат:**  
Статус заявки обновляется, комментарий сохраняется.

## TC-06. Вход под администратором

**Шаги:**

1. Выйти из учетной записи сотрудника.
2. Войти под администратором.
3. Открыть административную панель.

**Ожидаемый результат:**  
Администратор видит статистику, пользователей, категории и отчетность.

## TC-07. Скачивание отчета XLSX

**Шаги:**

1. Войти под администратором.
2. Нажать кнопку скачивания XLSX.

**Ожидаемый результат:**  
Файл отчета по заявкам скачивается на компьютер.
