# Описание структуры базы данных

Документ описывает основные таблицы базы данных веб-информационной системы StudentRequest и их назначение.

## Назначение базы данных

База данных используется для хранения информации о пользователях, заявках, категориях, комментариях, прикрепленных файлах, истории изменения статусов и действиях пользователей.

Централизованное хранение данных позволяет обеспечить:

- сохранность заявок студентов;
- быстрый поиск информации;
- контроль статусов обработки;
- хранение истории взаимодействия;
- формирование отчетности.

## Основные таблицы

### users

Таблица предназначена для хранения учетных записей пользователей системы.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор пользователя |
| full_name | ФИО пользователя |
| email | адрес электронной почты для входа |
| password_hash | хеш пароля |
| role | роль пользователя |
| is_active | признак активности учетной записи |
| created_at | дата создания учетной записи |

### students

Таблица содержит дополнительные сведения о студентах.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор записи |
| user_id | связь с таблицей users |
| group_name | учебная группа |
| course | курс |
| direction | направление подготовки |

### employees

Таблица содержит сведения о сотрудниках структурного подразделения.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор записи |
| user_id | связь с таблицей users |
| position | должность |
| department | подразделение |

### request_categories

Таблица хранит категории заявок.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор категории |
| name | название категории |
| description | описание категории |
| is_active | признак активности категории |

### requests

Основная таблица системы, предназначенная для хранения заявок студентов.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор заявки |
| student_id | студент, создавший заявку |
| category_id | категория заявки |
| title | тема заявки |
| description | описание обращения |
| status | текущий статус заявки |
| priority | приоритет заявки |
| created_at | дата создания |
| updated_at | дата последнего изменения |

### request_files

Таблица хранит сведения о файлах, прикрепленных к заявкам.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор файла |
| request_id | заявка, к которой прикреплен файл |
| file_name | имя файла |
| file_path | путь к файлу |
| uploaded_at | дата загрузки |

### request_comments

Таблица предназначена для хранения комментариев к заявкам.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор комментария |
| request_id | заявка, к которой относится комментарий |
| user_id | пользователь, оставивший комментарий |
| comment_text | текст комментария |
| created_at | дата создания комментария |

### status_history

Таблица хранит историю изменения статусов заявок.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор записи |
| request_id | заявка |
| old_status | предыдущий статус |
| new_status | новый статус |
| changed_by | пользователь, изменивший статус |
| changed_at | дата изменения статуса |

### activity_log

Таблица предназначена для фиксации действий пользователей.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор записи |
| user_id | пользователь, выполнивший действие |
| action | описание действия |
| entity_type | тип объекта |
| entity_id | идентификатор объекта |
| created_at | дата выполнения действия |

### settings

Таблица используется для хранения настроек системы.

Основные поля:

| Поле | Назначение |
|---|---|
| id | уникальный идентификатор настройки |
| setting_key | ключ настройки |
| setting_value | значение настройки |

## Основные связи между таблицами

- один пользователь может быть студентом или сотрудником;
- один студент может создать несколько заявок;
- одна заявка относится к одной категории;
- одна заявка может иметь несколько комментариев;
- одна заявка может иметь несколько прикрепленных файлов;
- одна заявка может иметь несколько записей истории статусов;
- действия пользователей фиксируются в журнале activity_log.

## Вывод

Структура базы данных StudentRequest обеспечивает хранение основных сущностей системы и позволяет реализовать учет заявок, разграничение ролей, обработку обращений, хранение комментариев, прикрепление файлов и формирование отчетности.
