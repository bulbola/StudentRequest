# Матрица ролей и прав доступа

Документ описывает роли пользователей веб-информационной системы StudentRequest и их права доступа к основным функциям.

## Роли пользователей

В системе предусмотрены три основные роли:

1. Студент
2. Сотрудник подразделения
3. Администратор

## Описание ролей

### Студент

Студент является основным пользователем системы, который создает заявки и отслеживает ход их обработки.

Основные возможности студента:

- вход в личный кабинет;
- создание новой заявки;
- выбор категории заявки;
- прикрепление файлов к заявке;
- просмотр списка своих заявок;
- просмотр статуса обработки;
- просмотр комментариев сотрудника.

### Сотрудник подразделения

Сотрудник подразделения отвечает за обработку заявок студентов.

Основные возможности сотрудника:

- вход в панель сотрудника;
- просмотр поступивших заявок;
- открытие карточки заявки;
- изменение статуса заявки;
- добавление комментариев;
- закрытие обработанных заявок;
- просмотр истории обработки.

### Администратор

Администратор управляет справочниками, пользователями и настройками системы.

Основные возможности администратора:

- вход в административную панель;
- управление пользователями;
- управление категориями заявок;
- просмотр отчетности;
- просмотр журнала действий;
- контроль работы системы.

## Матрица доступа

| Функция системы | Студент | Сотрудник | Администратор |
|---|---:|---:|---:|
| Вход в систему | Да | Да | Да |
| Просмотр главной страницы | Да | Да | Да |
| Просмотр справки | Да | Да | Да |
| Создание заявки | Да | Нет | Нет |
| Просмотр своих заявок | Да | Нет | Нет |
| Просмотр всех заявок | Нет | Да | Да |
| Обработка заявки | Нет | Да | Нет |
| Изменение статуса заявки | Нет | Да | Да |
| Добавление комментария | Нет | Да | Да |
| Прикрепление файла к заявке | Да | Нет | Нет |
| Управление пользователями | Нет | Нет | Да |
| Управление категориями | Нет | Нет | Да |
| Просмотр отчетности | Нет | Да | Да |
| Просмотр журнала действий | Нет | Нет | Да |

## Принцип разграничения доступа

Разграничение доступа используется для защиты данных и предотвращения выполнения пользователем действий, не относящихся к его роли.

Студент работает только со своими заявками.  
Сотрудник обрабатывает заявки студентов, но не управляет пользователями.  
Администратор выполняет настройку системы и контролирует справочники.

## Вывод

Использование ролевой модели позволяет обеспечить управляемость системы, повысить безопасность данных и разделить ответственность между участниками процесса обработки заявок.
