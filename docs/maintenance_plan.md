# План сопровождения проекта

Документ описывает возможные действия по сопровождению и развитию StudentRequest после завершения преддипломной практики.

## Регулярная проверка проекта

Для сопровождения проекта рекомендуется:

- проверять запуск приложения;
- проверять установку зависимостей;
- запускать тесты;
- проверять работу основных ролей;
- обновлять документацию при изменении функционала.

## Проверка данных

Необходимо контролировать:

- корректность пользователей;
- актуальность категорий заявок;
- сохранность заявок;
- корректность статусов;
- наличие комментариев и истории обработки.

## Обновление интерфейса

При сопровождении интерфейса можно улучшать:

- отображение статусов;
- таблицы заявок;
- форму создания заявки;
- страницу обработки заявки;
- административные страницы.

## Обновление документации

При изменении проекта нужно обновлять:

- README.md;
- инструкцию запуска;
- сценарий демонстрации;
- тестовые сценарии;
- скриншоты интерфейса.

## Возможное развитие

В дальнейшем можно добавить:

- фильтрацию заявок;
- поиск по заявкам;
- уведомления;
- расширенную отчетность;
- редактирование категорий через интерфейс;
- управление пользователями через интерфейс.

## Вывод

План сопровождения помогает поддерживать проект в рабочем состоянии и развивать его после сдачи.
