# Критерии приемки проекта

Документ описывает критерии, по которым можно считать проект StudentRequest готовым к демонстрации и сдаче.

## Функциональные критерии

Проект считается готовым, если:

- реализована авторизация пользователей;
- реализовано разделение доступа по ролям;
- студент может создавать заявки;
- сотрудник может обрабатывать заявки;
- администратор может просматривать пользователей и категории;
- администратор может выгружать отчет XLSX;
- система отображает справочную информацию;
- данные сохраняются в базе данных.

## Критерии интерфейса

Интерфейс считается готовым, если:

- основные страницы имеют единый стиль;
- главная страница содержит описание системы;
- страница входа содержит тестовые учетные записи;
- личный кабинет студента отображает список заявок;
- панель сотрудника отображает очередь заявок;
- панель администратора отображает основные показатели;
- страницы пользователей и категорий оформлены таблицами.

## Критерии документации

Документация считается подготовленной, если:

- в README есть описание проекта;
- есть инструкция локального запуска;
- есть тестовые учетные записи;
- есть описание ролей;
- есть сценарий демонстрации;
- есть скриншоты интерфейса;
- есть чек-листы проверки.

## Критерии репозитория

Репозиторий считается подготовленным, если:

- он доступен по публичной ссылке;
- история коммитов отражает поэтапную разработку;
- исходный код структурирован по папкам;
- документация находится в папке `docs`;
- тесты находятся в папке `tests`.

## Вывод

Выполнение данных критериев подтверждает готовность проекта к демонстрации и использованию в отчете по преддипломной практике.
