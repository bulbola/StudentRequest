# Тест-план StudentRequest

## Объект тестирования
Веб-информационная система учета и обработки заявок студентов.

## Проверяемые функции
1. Авторизация пользователей.
2. Разграничение доступа по ролям.
3. Создание заявки студентом.
4. Загрузка файла к заявке.
5. Просмотр карточки заявки.
6. Обработка заявки сотрудником.
7. Смена статуса и добавление комментариев.
8. Управление пользователями и категориями администратором.
9. Формирование XLSX-отчета.
10. Просмотр справки по системе.

## Критерии прохождения
Фактический результат совпадает с ожидаемым; данные сохраняются в БД; доступ к разделам соответствует роли пользователя; отчет XLSX формируется без ошибок.

## Критерии остановки
Критическая ошибка авторизации, невозможность создать заявку, ошибка подключения к БД, потеря данных после сохранения.
