# План тестирования

Проверка выполняется через pytest и coverage.

```bash
python -m pytest
coverage run -m pytest
coverage report -m
```

В текущей версии реализовано 17 автоматических тестов. Проверяются:

| Код | Сценарий | Ожидаемый результат |
|---|---|---|
| TC-01 | Создание приложения | Фабрика `create_app` возвращает приложение |
| TC-02 | Доступ студента к `/admin/` | HTTP 403 |
| TC-03 | Справка `/help` | HTTP 200, есть инструкция по SLA и повторной подаче |
| TC-04 | Создание заявки с коротким описанием | HTTP 400 |
| TC-05 | Создание корректной заявки | Созданы запись заявки, история, журнал и уведомления |
| TC-06 | Ограничение сотрудника по категории | HTTP 403 для чужой категории |
| TC-07 | Одновременное взятие заявки | Второй сотрудник получает HTTP 409 |
| TC-08 | Обработка без блокировки | HTTP 409 |
| TC-09 | Отклонение без причины | Статус не меняется |
| TC-10 | Отклонение с причиной | Статус `rejected`, причина сохранена |
| TC-11 | Повторная подача | Статус возвращается в `created`, SLA пересчитана |
| TC-12 | Руководитель | Статистика доступна, админ-панель недоступна |
| TC-13 | Администратор | Создание пользователя и категории |
| TC-14 | Проверка SLA | Просроченная заявка эскалируется, создаются уведомления |
| TC-15 | Проверка страницы справки | /help возвращает 200 и содержит инструкцию |
| TC-16 | Проверка CRUD пользователей | Администратор просматривает и изменяет пользователей |
| TC-17 | Проверка CRUD категорий | Администратор просматривает и изменяет категории |

Текущий отчет покрытия сохранен в `docs/coverage_report.txt`: 17 тестов успешно, совокупное покрытие 84%.
