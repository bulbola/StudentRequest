# Заметки по безопасности

Документ описывает основные меры безопасности, применяемые в проекте StudentRequest.

## Авторизация

Для доступа к личным кабинетам используется авторизация пользователей по email и паролю.

После входа пользователь получает доступ только к тем разделам, которые соответствуют его роли.

## Роли пользователей

В системе используются три роли:

- студент;
- сотрудник подразделения;
- администратор.

Ролевое разделение позволяет ограничить доступ к функциям системы.

## Пароли

Пароли пользователей не должны храниться в открытом виде.  
Для хранения паролей используется хеширование.

## Ограничение доступа

Студент работает только со своими заявками.  
Сотрудник работает с очередью заявок.  
Администратор имеет доступ к справочникам, пользователям и отчетности.

## Загружаемые файлы

При загрузке файлов необходимо контролировать:

- имя файла;
- расширение файла;
- путь сохранения;
- отсутствие выполнения загруженных файлов как кода.

## Вывод

Использование авторизации, ролей и ограничения доступа повышает защищенность системы и снижает риск несанкционированного доступа к данным.
