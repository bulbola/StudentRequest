# Заметки по резервному копированию данных

Документ описывает рекомендации по резервному копированию данных в проекте StudentRequest.

## Назначение резервного копирования

Резервное копирование необходимо для сохранения данных системы и восстановления работы при ошибках.

В StudentRequest данные хранятся в базе данных SQLite при локальном запуске.

## Что нужно сохранять

Для восстановления проекта необходимо сохранять:

- исходный код приложения;
- базу данных;
- загруженные пользователями файлы;
- файл requirements.txt;
- документацию проекта;
- скриншоты и материалы отчета.
