# Риски проекта

Документ описывает возможные риски при разработке и использовании системы StudentRequest.

## Технические риски

### Ошибки запуска

Проект может не запуститься на другом компьютере из-за отсутствия Python, зависимостей или неправильной настройки окружения.

Способ снижения риска:

- использовать requirements.txt;
- описать команды запуска в README;
- добавить чек-лист локального запуска.

### Ошибки базы данных

База данных может отсутствовать или быть не инициализирована.

Способ снижения риска:

- использовать команду init-db;
- использовать команду seed-db;
- описать порядок запуска в документации.

## Пользовательские риски

### Неверный ввод данных

Пользователь может заполнить заявку неполно или некорректно.

Способ снижения риска:

- использовать обязательные поля;
- добавить подсказки на форме создания заявки;
- применять категории и приоритеты.

### Ошибки при выборе роли

Пользователь может не понимать различия между ролями.

Способ снижения риска:

- описать роли в справке;
- добавить демонстрационные учетные записи;
- описать сценарий демонстрации.

## Организационные риски

### Недостаток материалов для защиты

Для демонстрации могут потребоваться скриншоты, ссылка на GitHub и история коммитов.

Способ снижения риска:

- подготовить скриншоты;
- добавить их в репозиторий;
- оформить README;
- подготовить отчет.

## Вывод

Анализ рисков помогает заранее проверить слабые места проекта и подготовить материалы для защиты.
