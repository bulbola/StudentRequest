# Пользовательские сценарии

Документ описывает основные пользовательские сценарии работы с системой StudentRequest.

## Сценарий студента

1. Студент открывает главную страницу системы.
2. Переходит на страницу входа.
3. Вводит email и пароль.
4. Попадает в личный кабинет.
5. Просматривает список своих заявок.
6. Создает новую заявку.
7. Выбирает категорию обращения.
8. Указывает тему, описание и приоритет.
9. Отправляет заявку.
10. Проверяет статус обработки.

## Сценарий сотрудника

1. Сотрудник входит в систему.
2. Открывает панель сотрудника.
3. Просматривает очередь заявок.
4. Выбирает заявку для обработки.
5. Изучает описание обращения.
6. Меняет статус заявки.
7. Добавляет комментарий.
8. Сохраняет изменения.

## Сценарий администратора

1. Администратор входит в систему.
2. Открывает административную панель.
3. Просматривает статистику.
4. Открывает список пользователей.
5. Открывает список категорий.
6. При необходимости скачивает отчет XLSX.
7. Контролирует состояние системы.

## Общий жизненный цикл заявки

1. Создание заявки студентом.
2. Поступление заявки в очередь сотрудника.
3. Рассмотрение заявки сотрудником.
4. Изменение статуса.
5. Добавление комментария.
6. Просмотр результата студентом.
7. Закрытие заявки.
