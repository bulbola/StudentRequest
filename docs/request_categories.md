# Категории заявок и приоритеты

Документ описывает категории заявок, используемые в веб-информационной системе StudentRequest, а также модель приоритетов обработки обращений студентов.

## Назначение категорий

Категории заявок используются для классификации обращений студентов и упрощения их обработки сотрудниками структурного подразделения Университета.

Использование категорий позволяет:

- быстрее определить тип обращения;
- назначить ответственное лицо;
- анализировать статистику обращений;
- формировать отчеты по направлениям работы;
- контролировать наиболее частые причины обращений.

## Основные категории заявок

| Код | Категория | Описание |
|---|---|---|
| DOC | Документы | Запросы, связанные со справками, заявлениями, отчетными материалами и иными документами |
| PRACTICE | Практика | Вопросы по прохождению практики, индивидуальному заданию, отчету и срокам загрузки материалов |
| STUDY | Учебный процесс | Вопросы по расписанию, дисциплинам, консультациям и взаимодействию с преподавателями |
| TECH | Техническая поддержка | Проблемы с доступом, загрузкой файлов, отображением страниц и работой системы |
| OTHER | Другое | Обращения, которые не относятся к основным категориям |

## Приоритеты заявок

Для заявок предусмотрены три уровня приоритета.

| Приоритет | Описание | Пример |
|---|---|---|
| Низкий | Заявка не требует срочной обработки | общий вопрос, уточнение информации |
| Средний | Заявка требует обработки в стандартные сроки | вопрос по документам или учебному процессу |
| Высокий | Заявка требует оперативного рассмотрения | проблема со сроками сдачи, невозможность загрузить отчет |

## Связь категории и приоритета

Категория определяет предмет обращения, а приоритет определяет срочность обработки.

Например, заявка категории «Практика» может иметь средний приоритет, если студент уточняет порядок оформления отчета. Однако при невозможности загрузить отчет в установленный срок такая заявка может получить высокий приоритет.

## Использование в системе

В StudentRequest категория выбирается студентом при создании заявки.  
Приоритет может быть назначен автоматически или выбран пользователем в зависимости от реализации формы.

Сотрудник подразделения использует категорию и приоритет для сортировки очереди заявок и определения порядка обработки.

## Применение в отчетности

Категории и приоритеты могут использоваться при формировании отчетов:

- количество заявок по категориям;
- количество заявок по приоритетам;
- количество закрытых и незакрытых заявок;
- анализ наиболее частых обращений;
- контроль нагрузки на сотрудников.

## Вывод

Использование категорий и приоритетов повышает управляемость процесса обработки заявок и позволяет сделать работу структурного подразделения более прозрачной и контролируемой.
